# 多服面板合并（主/子服、代理与配对）

本功能用于将多个 WebUI 实例“合并到一个入口”进行管理：**浏览器始终连接主服**，主服根据你选择的目标服务器，把大部分 `/api/`* 请求**反向代理**到对应子服，从而做到“一个面板看多台服”。

> 关键原则
>
> - **权限判定在主服**：是否允许执行某个操作由主服的登录态与权限决定。
> - **Token 有效性在子服**：主服代理到子服时，会携带 `X-Panel-Token`，子服只负责校验这个 token 是否有效（以及可选的来源 IP 限制）。
> - **子服可继续允许管理员直登**：在 allow-direct 的思路下，子服仍保留原有 cookie 登录；同时额外允许主服 token 访问。

---

## 概念与模式

### 1) 主服（master）

- 对外提供 WebUI 访问入口（浏览器只访问主服）
- 维护子服列表（子服名称、面板地址、token、启用状态等）
- 根据请求头 `X-Target-Server` 决定是否代理到子服

### 2) 子服（slave）

- 仍可提供本地网页登录（可选）
- 维护允许主服访问的 token 列表（`allowed_tokens`，可启用/禁用）
- 在接收到 `X-Panel-Token` 时校验 token（以及可选的来源 IP 白名单）

## 快捷模式（推荐）

快捷模式面向“无需手写 token/配置”的常规使用场景。

### 主服：连接子服

1. 打开 **WebUI 设置 → 多服面板合并 → 快捷模式**
2. 选择角色为 **主服**
3. 在“子服列表”输入：
  - 子服名称
  - 子服面板地址（例如 `http://127.0.0.1:8001`）
4. 点击 **连接**
5. 主服会进入“等待子服确认连接…”状态，并自动轮询结果（最长 5 分钟）

### 子服：接受连接（5 分钟窗口）

1. 打开 **WebUI 设置 → 多服面板合并 → 快捷模式**
2. 选择角色为 **子服**
3. 点击 **开始接受连接**
  - 将开启一个 5 分钟窗口
  - 收到第一个连接请求后会自动关闭窗口
4. 当出现“待确认连接”时，你会看到来源 IP
5. 点击 **接受**
  - 子服会自动生成一个 token，并保存到本地配置
  - 主服会自动拿到 token 并保存子服信息到主服配置

完成后，回到主服侧边栏即可切换服务器数据源（无需整页刷新）。

## 配置模式（高级）

配置模式适合需要精确控制字段（例如手动维护 token、手工编辑地址、调整启用状态等）的场景。

### 主服侧（panel_slaves）

典型字段（概念说明）：

- `id`: 子服标识（用于 `X-Target-Server`）
- `name`: 显示名称
- `base_url`: 子服面板地址（不包含末尾 `/api`）
- `token`: 主服访问子服所需的 `X-Panel-Token`
- `enabled`: 是否启用该子服
- `verify_tls`: 代理访问时是否校验证书（HTTPS 场景）

### 子服侧（panel_master.allowed_tokens）

典型字段（概念说明）：

- `token`: 允许主服访问的 token 值
- `enabled`: 是否启用该 token
- `name`: 备注名称（用于识别哪个主服）

## 代理规则（重要）

### 什么时候会代理

当满足以下条件时，主服会把请求代理到子服：

- 请求路径属于 `/api/`*
- 且请求头（或 query）中指定了非 `local` 的目标服务器：
  - Header：`X-Target-Server: <server_id>`
  - Query：`?serverId=<server_id>`（兼容用法）
- 且主服自身配置 `panel_role=master`

### 哪些接口不会被代理（主服本地处理）

为了避免 cookie/登录态、握手、以及大数据传输产生混乱，以下接口始终在主服本地处理：

- `/api/login`、`/api/logout`、`/api/checkLogin`
- `/api/servers`
- `/api/panel_merge_config`
- `/api/langs`
- `/api/online-plugins`
- `/api/pairing/*`

> 说明：在线插件页的“安装/操作”会走代理到子服，但大列表数据本身保持本地，以减少无意义传输。

## 常见问题（FAQ）

### 1) 为什么侧边栏看不到服务器切换？

- 只有当主服 `/api/servers` 返回至少一个启用的子服时，侧边栏才会显示服务器切换。
- 如果你刚完成配对，请确认：
  - 主服已保存子服（连接成功提示出现）
  - 子服条目 `enabled=true`

### 2) 为什么访问子服接口提示未登录/401？

- 说明子服没有接受到有效的 `X-Panel-Token`，或 token 未在子服的 `allowed_tokens` 中启用。
- 你可以在子服的配置模式检查 `allowed_tokens` 列表是否存在且启用。

### 3) 子服是否还能给管理员直接登录？

可以。子服仍保留原 cookie 登录逻辑；`X-Panel-Token` 是额外通道，仅用于主服代理访问。
